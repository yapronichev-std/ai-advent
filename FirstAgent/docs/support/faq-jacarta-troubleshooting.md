# FAQ: JC-Mobile SDK Android — Решение проблем

## Ошибка "Библиотека не найдена" (UnsatisfiedLinkError)

**Причина:** Нативные библиотеки (.so) не найдены или не соответствуют архитектуре устройства.

**Решение:**
1. Проверьте, что папки `arm64-v8a` и `armeabi-v7a` скопированы в `app/src/main/jniLibs/`
2. Убедитесь, что внутри папок присутствуют все 4 файла: `libjcPKCS11-2.so`, `libgti_jni.so`, `libjnidispatch.so`, `libjcpcsclite.so`
3. Проверьте архитектуру тестового устройства (32 или 64 бита)
4. Для 64-битных устройств обязательна папка `arm64-v8a`, для 32-битных — `armeabi-v7a`

## Ошибка при сборке Gradle: "ANDROID_SDK_ROOT not set"

**Причина:** Не задана переменная окружения ANDROID_SDK_ROOT.

**Решение:**
```bash
export ANDROID_SDK_ROOT="/Users/username/Library/android/android-sdk"
```
Убедитесь, что путь указывает на реальную директорию с Android SDK.

## Не удается найти токен или смарт-карту

**Причина:** Устройство не подключено или JaCarta Service не запущен.

**Решение:**
1. Убедитесь, что приложение **JaCarta Service** установлено из Google Play
2. Для NFC-подключения: убедитесь, что NFC включен на устройстве, поднесите смарт-карту к NFC-антенне
3. Для ридера JCR721: подключите ридер через USB-OTG, проверьте индикацию
4. Вызовите `C.GetSlotList` для проверки доступных слотов

## Ошибка "PIN-код неверный" или PIN-код заблокирован

**Причина:** Исчерпаны попытки ввода PIN-кода.

**Решение:**
1. Используйте PUK-код для разблокировки:
   - Для Криптотокен 2 ЭП: `GOST2_unlockPinUser`
   - Для Laser: `PKI_pinUserUnblock` или `Extensions.JC_PKI_UnlockUserPIN`
   - Для Криптотокен: `GOST_pinUserUnblock`
2. Если PUK-код также заблокирован — требуется переинициализация устройства через ПК Единый Клиент JaCarta

## Ошибка при сборке common.jar

**Причина:** Неправильная настройка Gradle или отсутствие JDK 11.

**Решение:**
1. Проверьте версию Java: `java -version` (должна быть 11)
2. Перейдите в директорию `common` и выполните: `./gradlew makeJar`
3. Результат `common.jar` появится в папке `libs/`
4. Если Gradle не запускается — проверьте права на выполнение: `chmod +x gradlew`

## Не удается запустить пример на устройстве

**Причина:** Устройство не в режиме разработчика или не настроена отладка по USB.

**Решение:**
1. На устройстве Android: Настройки → О телефоне → 7 раз нажать "Номер сборки"
2. Вернуться в Настройки → Для разработчиков → Включить "Отладку по USB"
3. Включить "Установку по USB"
4. Устройство должно быть разблокировано при запуске
5. Подтвердить доверие к компьютеру при первом подключении

## Не установлено приложение JaCarta Service

**Причина:** JaCarta Service — обязательный компонент для работы SDK.

**Решение:**
1. Откройте Google Play на устройстве
2. Найдите "JaCarta Service" (разработчик: Аладдин Р.Д.)
3. Установите приложение
4. Предоставьте необходимые разрешения (NFC, доступ к USB)

## Ошибка 403 или "Access Denied" при вызове функций PKCS #11

**Причина:** Не выполнена аутентификация (C.Login) или сессия закрыта.

**Решение:**
1. Убедитесь, что сессия открыта: `C.OpenSession`
2. Выполните вход с PIN-кодом: `C.Login(session, CKU_USER, pin)`
3. Проверьте, что PIN-код пользователя установлен (для нового токена требуется инициализация)
4. После завершения работы корректно закройте сессию: `C.Logout` → `C.CloseSession`

## Различия между 32-битной и 64-битной архитектурой

**Важно:** Для поддержки обеих архитектур необходимо скопировать обе папки (`arm64-v8a` и `armeabi-v7a`) в `jniLibs`. Android автоматически выберет нужную архитектуру при установке APK.
